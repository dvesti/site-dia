Hugo uses Markdown for its simple content format. However, there are a lot of things that Markdown doesn‚Äôt support well. You could use pure HTML to expand possibilities.

But this happens to be a bad idea. Everyone uses Markdown because it‚Äôs pure and simple to read even non-rendered. You should avoid HTML to keep it as simple as possible.

To avoid this limitations, Hugo created shortcodes. A shortcode is a simple snippet that can generate reasonable HTML code and conforms to Markdown’s design philosophy.

Hugo ships with a set of predefined shortcodes that represent very common usage. These shortcodes are provided for author convenience and to keep your markdown content clean.
